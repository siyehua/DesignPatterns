#设计模式入门(策略模式)
策略模式定义了算法组,分辨封装起来,让他们之间可以相互调换,此模式让算法的变化独立于使用算法的客户.

##要点
 * 良好的OO设计必须具备可复用,可扩充,可维护三个特性
 * 模式可以让我们建造出具有良好OO设计质量的系统
 * 模式被认为是历经考验的OO设计经验
 * 模式不是代码,而是针对设计问题的通用解决方案.你可以把它们应用到特定的应用中
 * 大多数的模式和原则,都着眼于软件变化的主题
 * 我们常常把系统中会变化的部分抽出来封装.
 * 模式让开发人员之间有共享的语言,能够最大化沟通的价值.
